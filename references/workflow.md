# 工作流程

## 阶段 0：用户输入与决策

**触发条件**：用户在项目目录中调用 diff-review-zh skill

### 步骤 0.1：询问比较范围

必须询问用户要比较哪些修改：

```
请选择要审查的代码范围：

[1] 未提交更改 vs 上次提交（git diff HEAD）
[2] 当前分支 vs 目标分支（git diff main...HEAD）
[3] 指定两次提交对比（自定义范围）
[4] 指定 PR/MR 链接

请输入选项（1-4）：
```

### 步骤 0.2：根据选择获取 diff

| 选项 | 操作 |
|------|------|
| 1 | 运行 `git diff HEAD` |
| 2 | 运行 `git diff main...HEAD`（或用户指定的目标分支） |
| 3 | 询问两次 commit SHA，运行 `git diff SHA1..SHA2` |
| 4 | 从 PR API 获取 diff |

### 步骤 0.3：确认 diff 范围

显示获取到的修改概览，询问用户确认：

```
找到以下修改：
- 文件数：9
- 添加行数：+1967
- 删除行数：-59
- 主要修改：runner.py, registry.py

确认生成审查报告？(y/n)：
```

---

## 阶段 1：分析修改（并行执行）

读取 diff 后，并行分析：

### 任务 1.1：统计信息
- 计算添加/删除行数
- 统计修改文件数
- 分类：新增/修改/删除

### 任务 1.2：提取修改动机
- 查找 commit message
- 查找 PR 描述
- 识别关联的 issue/ticket

### 任务 1.3：识别关键修改
- 找出核心逻辑变更
- 识别新增函数/方法
- 标记重要删除

### 任务 1.4：构建术语表
- 识别专业术语
- 准备中文解释
- 标记需要 tooltip 的词汇

---

## 阶段 2：内容规划

### 决策点 2.1：选择展示模式

根据修改规模选择：

| 修改规模 | 展示模式 |
|---------|---------|
| ≤ 3 文件，≤ 100 行 | 单页：概览 + 代码详情 |
| 3-10 文件，或 100-500 行 | 多页：概览 + 运行时 + 代码详情 |
| ≥ 10 文件，或 ≥ 500 行 | 多页：完整导航 + 按模块分组 |

### 决策点 2.2：规划模块

基于修改类型决定包含哪些模块：

**必须包含**：
- 项目总览（文件树 + 统计）
- 代码修改详情（每个修改文件）

**条件包含**：
- 运行时对话：复杂业务逻辑修改
- 数据流追踪：涉及多组件交互的修改
- 变更时间线：多步骤/多 commit 的修改
- 状态对比：API/接口变更

---

## 阶段 3：生成内容（顺序执行）

### 步骤 3.1：创建输出目录

```bash
mkdir -p diff-review-{timestamp}/
```

### 步骤 3.2：生成 HTML 结构

使用模板生成基础 HTML：
1. 复制设计系统 CSS（内联）
2. 复制交互 JS（内联）
3. 生成模块结构

### 步骤 3.3：填充内容

对每个修改文件：
1. 生成代码对比块
2. 撰写白话解释
3. 应用语法高亮

### 步骤 3.4：添加交互元素

- 初始化导航点
- 设置对话/流动画
- 绑定事件处理器

---

## 阶段 4：质量检查

### 检查点 4.1：设计合规

- [ ] 背景颜色正确
- [ ] 字体加载正确
- [ ] 代码格式正确（无空行，缩进保留）
- [ ] 响应式正常

### 检查点 4.2：内容完整

- [ ] 每个修改都有解释
- [ ] 解释了"为什么"
- [ ] 术语有定义
- [ ] 文件路径正确

### 检查点 4.3：功能测试

- [ ] 导航可工作
- [ ] 代码块可滚动
- [ ] 键盘导航可用
- [ ] 移动端正常

## 决策流程图

```
启动
  ↓
询问比较范围 ←── 用户选择 ──→ 获取 diff
  ↓                              ↓
显示概览 ←────── 确认 ───────→ 分析修改（并行）
  ↓
选择展示模式
  ↓
规划模块内容
  ↓
生成 HTML（顺序）
  ↓
质量检查
  ↓
交付文件
```

---

## 并行 vs 顺序策略

### 并行执行（性能优化）
- 分析阶段的所有任务（统计、动机、术语）
- 多个文件的代码高亮处理

### 顺序执行（依赖关系）
- 必须先获取 diff 才能分析
- 必须先规划模块才能生成内容
- 必须先生成 HTML 才能质量检查
